Routing Mode
TUN mode routes all traffic transparently through the SOCKS5 proxy via a virtual network interface.

After reconnecting the proxy, stale DNS entries can point at a previous tunnel session. On Windows the app flushes the system DNS cache before starting TUN mode; browser-internal DNS caches may still need a browser restart or a manual clear:
Firefox: about:networking#dns → Clear DNS Cache
In system mode, the app starts an embedded local SOCKS5 adapter on 127.0.0.1:1081 and points the operating system proxy settings at it.

The local adapter forwards to your configured upstream SOCKS5 proxy and handles username/password authentication there. On Windows the app notifies running WinINET-aware applications after changing proxy settings; DNS behavior can still vary by application.

Firefox: Set Settings → General → Network Settings → Use system proxy settings so Firefox uses the embedded local adapter.
